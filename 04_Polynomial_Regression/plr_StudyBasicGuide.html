Polynomial Regression

Complete Study Guide | دليل المذاكرة الشامل

Phase 1: Importing & Preparing Data 📦

import numpy as np
import pandas as pd

dataset = pd.read_csv('Position_Salaries.csv')
X = dataset.iloc[:, 1:-1].values  # Skip the text column (Index 0)
y = dataset.iloc[:, -1].values
English: We load the dataset. Since column 0 is text (e.g., "CEO") and column 1 represents those exact levels numerically (1 to 10), we skip column 0 and slice only column 1 for X to save encoding time.
عربي: بنقرا ملف الداتا. وبما إن العمود الأول عبارة عن كلام بيعبر عن نفس رتب العمود التاني بالأرقام، بنطنش العمود الأول وبناخد العمود التاني بس كـ X عشان نوفر على نفسنا خطوة الـ Encoding.

Phase 2: Transforming Features (The Gymnastics) 🤸

from sklearn.preprocessing import PolynomialFeatures
poly_reg = PolynomialFeatures(degree=4)
X_poly = poly_reg.fit_transform(X)
English: There is no "PolynomialRegression" class. Instead, we use PolynomialFeatures to clone and power our single feature X, generating a new matrix X_poly containing $X^1, X^2, X^3, X^4$.
عربي: مفيش كلاس جاهز اسمه PolynomialRegression. بنستخدم PolynomialFeatures عشان ننسخ ونربع ونكعب العمود الأصلي X ونعمل عواميد جديدة فيها أس 2 وأس 3 وأس 4. العواميد دي هي اللي بتخلي الخط يلتوي!

Phase 3: Fitting the Linear Model 🚂

from sklearn.linear_model import LinearRegression
lin_reg_2 = LinearRegression()
lin_reg_2.fit(X_poly, y)
English: Now that our single feature is transformed into powered features, we feed them into the standard LinearRegression model (called lin_reg_2). Under the hood, it's just Multiple Linear Regression!
عربي: بعد ما حولنا العمود الواحد لمجموعة عواميد فيها أسس مختلفة، بنرميها للموديل العادي بتاعنا LinearRegression (وسميناه lin_reg_2) عشان يدرب عليها. في الكواليس، الموديل بيتعامل معاها كأنها انحدار خطي متعدد!

Phase 4: Visualizing with High Resolution (Smoother Curve) 📈

X_grid = np.arange(min(X), max(X), 0.1)

عربي: لو رسمنا الكيرف بالنقط الأساسية بس، الخط هيطلع مكسر ومش ناعم. عشان نخليه انسيابي، بنستخدم np.arange بخطوة 0.1 عشان نولد 90 نقطة في النص (1.0، 1.1، 1.2...). بنحولها لـ 2D بـ reshape عشان الموديل يتوقع النقط دي كلها فيترسم كيرف ناعم ومثالي.

The Master Schema: 5 Questions for Polynomial Regression 🧠

1. The "5-Year-Old" Question (Core Logic)

Standard Linear Regression can only draw a rigid straight line. Polynomial Regression is its flexible cousin that has the power to bend the line to fit curved data.

Visual:

Linear (Fail):
  *   / *
    *   
  _ / * 
Polynomial (Win):
     *   *
   *   _/*
 _ * _/

2. Why is one column squared? ($x^2$)

Powers act as "mathematical accelerators."

If $x$ is small (Level 2), $x^2$ is only 4. If $x$ is large (Level 10), $x^2$ rocket-launches to 100.

This tells the model: "As Job Level increases, make the salary explode like a rocket, not climb like a staircase."

3. The "Data Diet" (Feature Scaling?)

Answer: NO.

Since it's still technically a linear model, the coefficients ($b_1, b_2$) shrink automatically to balance the huge squared numbers. However, if you use a crazy high degree (like $x^{10}$), you might need scaling to prevent your computer from crashing due to huge numbers.

4. The "Kryptonite" (Outliers)

Answer: Extremely Vulnerable.

Outliers are exponentially worse here because of the powers. An outlier at $x=10$ with `degree=4` becomes $10,000$ ($10^4$) in the math. The curve will warp completely just to reach that one crazy dot. Outliers must be cleaned first.

5. The "Control Knobs" (Overfitting Danger)

The Knob: The degree parameter in PolynomialFeatures(degree=4).

The Danger: If the degree is too low (e.g., 2), the line won't bend enough (Underfitting). If it's too high (e.g., 15), the line will wiggly-wiggle like a crazy snake just to touch every dot perfectly (Overfitting). Aim for a smooth, natural curve (usually degree 3 or 4).

Code Memorization Trick | تريكة للحفظ 💡

Remember the Two-Step Pipeline!
Scikit-learn doesn't have a "PolynomialRegression" class. To write the code from memory, just think of it as a 2-step recipe:

  1. Cook the powered variables: X_poly = PolynomialFeatures(degree=4).fit_transform(X)
  2. Feed them to the regular diner: LinearRegression().fit(X_poly, y)

2. Finding Model Accuracy (R-Squared)

Always output the exact R-Squared score to mathematically validate your beautiful curved line:

r2 = r2_score(y, lin_reg_2.predict(X_poly))
print(f"Polynomial R2 Accuracy: {r2 * 100:.2f}%")

عشان تثبت كفاءة الكيرف بلغة الأرقام الإحصائية، بنحسب الـ R-Squared اللي بيطلعلنا نسبة دقة الموديل في مطابقة البيانات.
